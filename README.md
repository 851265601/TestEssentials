# TestEssentials

Case Handling Tool 
 
 
Notes:  Stackoverflow 工程师需要提供 stackoverflow的 userid和自己的alias。userid可以在stackoverflow的个人主页面的link里面看到。 
Xamarin Official 的工程师需要提供 Official Forum的username 以及自己的alias 
 
Root Cause Analysis 
针对论坛上客户的任何错误的用法或者方向，我们都应该第一时间指出来问题的根本原因。 
 
 
Solution 
 
在提供了根本原因后我们可以提供相应的解决方案给客户，以下为提供解决方案的注意点： 
1.	当要贴codes作为解决方案的时候，严禁把没有任何描述的纯codes 作为解决方案，当codes很长的时候我们需要加上适当的注释让customer可以第一时间找出来你对他的codes做了什么改变。 
2.	请加上适当的图片，截图，以及gif动画修饰自己的解决方案。 
3.	当要share一篇文档中的某一部分的时候严禁只给一个文档link，然后让customer自己去找。应当指出这文档这种的哪一块是客户需要去看的当codes太多的时候，可以加上自己写的demo来让客户尝试运行。比较合理的方法是把demo传到自己的github，再把link 分享到你的answer中去。 
4.	如果没有找到solution，但是有workaround，我们可以先提供适当的workaround解决客户的燃眉之急。 
Notes：回答得不得体得answer会在Quality Review 中被Fail。 
 
Follow Up 
 
在客户回来reply了自己发的comment或者answer后我们要及时地follow up。不follow up的case 会在 Quality Review中被fail。
